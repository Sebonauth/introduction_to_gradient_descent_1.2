The Math Behind the Steps

Let's formalize the update rule. In each iteration, we update our parameter Î¸ (which can be a single value or a vector) using the following equation:

\[ \theta := \theta - \alpha\nabla C(\theta) \]

where:

  • \(\theta\) represents the parameter(s) we are trying to optimize.
  • \(\alpha\) is the learning rate.
  • \(\nabla C(\theta)\) is the gradient of the cost function at the current value of \(\theta\).

Gradient Descent Algorithm: Step-by-Step

The Gradient Descent algorithm follows these steps:

  1. Initialization: Choose a random starting value for \(\theta\).
  2. Iteration: Repeat the following steps until convergence or a maximum number of iterations is reached:
    • Calculate the Gradient: Compute \(\nabla C(\theta)\) at the current value of \(\theta\). This tells us the direction of the steepest ascent.
    • Update Parameters: Update \(\theta\) using the formula: \[ \theta := \theta - \alpha\nabla C(\theta) \]. This moves \(\theta\) in the direction of the steepest descent.
  3. Termination: Stop when a termination condition is met. This could be:
    • Convergence: The change in the cost function or the parameters becomes very small (below a predefined threshold \(\varepsilon\).
    • Iteration Limit: A maximum number of iterations (\(N\) is reached.
